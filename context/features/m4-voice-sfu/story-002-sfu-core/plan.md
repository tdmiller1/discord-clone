# Plan: Server — mediasoup SFU core (worker, router, voice rooms)

## Summary
Add `mediasoup` and a framework-agnostic `VoiceSfu` service (`server/src/voice/sfu.ts`) that owns one mediasoup `Worker` + Opus-only `Router`, plus an in-memory per-voice-channel room registry. The service exposes a small per-participant API (RTP caps, create/connect transport, produce, consume, pause/resume, list producers, close participant) that story 003's gateway will relay to clients. It is constructed and async-initialized in `buildApp` (which becomes `async`) and torn down on `onClose`. No gateway wiring, no DB, no persistence — that is later stories.

## Decisions resolved (from research "Decisions Made", all final)
- New `server/src/voice/` directory; the `VoiceRoom`/participant abstraction is kept **inline in `sfu.ts`** (small internal types + `Map<channelId, Room>` on `VoiceSfu`) — no separate `room.ts`, matching the one-small-file shape of `hub.ts`/`presence.ts`.
- Async init via a synchronous constructor + `async init()`, awaited in `buildApp`. `buildApp` becomes `async`; `index.ts` changes to `const app = await buildApp(config)` (it is already at top-level `await`).
- Two transports per participant (send + recv); consumers created `paused: true` (API exposes `resumeConsumer`).
- `mediaCodecs` = Opus only, `clockRate: 48000`, `channels: 2`.
- Transport `listenIps: [{ ip: "0.0.0.0", announcedIp: config.publicHost }]`, `enableUdp: true`, `enableTcp: true`, `preferUdp: true`. Worker RTC range from `config.rtcMinPort`/`config.rtcMaxPort`.
- Dockerfile: verification only — no structural change expected (build stage already has `python3 make g++`; runtime is glibc `node:24-bookworm-slim`). Add a comment noting mediasoup's prebuilt glibc worker requires staying on the Debian (non-alpine) base.

### Step 2: Create the `VoiceSfu` service
**File(s):** `server/src/voice/sfu.ts`
**Action:** create
**Description:** A plain exported class (no Fastify import; imports only `mediasoup` + its types and the `Config` type) mirroring `BroadcastHub`/`PresenceRegistry`: `#private` fields, terse method JSDoc, a module-level doc comment reaffirming "≤10 clients, one worker, no broker." It owns:
- `#worker: Worker | null`, `#router: Router | null`, set in `init()`.
- `#rooms: Map<number, VoiceRoom>` keyed by `channelId`.
- Internal `VoiceRoom` = `{ participants: Map<string, Participant> }`; internal `Participant` = `{ id; channelId; sendTransport: WebRtcTransport | null; recvTransport: WebRtcTransport | null; producer: Producer | null; consumers: Map<string, Consumer> }` (consumers keyed by remote producer id).

Constructor takes `config: Config` and stores `rtcMinPort`, `rtcMaxPort`, `publicHost` (no port binding in the constructor — keeps `new VoiceSfu(config)` synchronous and cheap, exactly like `new BroadcastHub()`).

Public API (signatures authoritative — see "New Types / Contracts"):
- `async init(): Promise<void>` — `createWorker({ rtcMinPort, rtcMaxPort, logLevel: "warn" })`, then `worker.createRouter({ mediaCodecs: [opus] })`. Idempotent guard (return early if already initialized). Stores worker/router. Attaches a `worker.on("died", ...)` log so a worker crash is observable (≤10 clients, single worker — log and rely on process restart; do not auto-respawn).
- `getRtpCapabilities(): RtpCapabilities` — returns `this.#router.rtpCapabilities` (throws if not initialized).
- `async createTransport(channelId, participantId, direction: "send" | "recv"): Promise<TransportParams>` — lazily creates the room + participant on first call; calls `router.createWebRtcTransport({ listenIps: [{ ip: "0.0.0.0", announcedIp: publicHost }], enableUdp: true, enableTcp: true, preferUdp: true })`; stores it on the participant under the direction (closing/replacing any prior transport for that direction to stay idempotent on re-join); returns `{ id, iceParameters, iceCandidates, dtlsParameters }`.
- `async connectTransport(channelId, participantId, direction, dtlsParameters): Promise<void>` — looks up the participant's transport for `direction`, `await transport.connect({ dtlsParameters })`.
- `async produce(channelId, participantId, rtpParameters): Promise<{ producerId: string }>` — `sendTransport.produce({ kind: "audio", rtpParameters })`; closes any pre-existing producer first (idempotent re-produce, never two mic tracks); stores it; returns its id.
- `async consume(channelId, consumerParticipantId, producerId, rtpCapabilities): Promise<ConsumeParams | null>` — guards `router.canConsume({ producerId, rtpCapabilities })` (returns `null` if false); `recvTransport.consume({ producerId, rtpParameters: ..., paused: true })`; stores under the producer id; returns `{ id, producerId, kind, rtpParameters }`.
- `async resumeConsumer(channelId, participantId, producerId): Promise<void>` — resumes the stored consumer (the post-handshake step).
- `pauseProducer(channelId, participantId): void` / `resumeProducer(channelId, participantId): void` — back mute; no-op if no producer.
- `listProducers(channelId, exceptParticipantId?): { participantId: string; producerId: string }[]` — every participant in the room with a live producer, optionally excluding one (the newcomer consuming existing producers / self-exclusion).
- `closeParticipant(channelId, participantId): { roomEmpty: boolean }` — closes the participant's send + recv transports (cascades to its producer + consumers), removes it from the room; if the room is now empty, closes/deletes the room and returns `roomEmpty: true` (the transition-reporting analogue of `PresenceRegistry.remove`'s `lastOffline`). Idempotent for unknown channel/participant.
- `async close(): Promise<void>` — closes all rooms (each participant's transports), then `router.close()` then `worker.close()`; clears state; idempotent (safe to call once on `onClose`, even if `init` never ran).

Internal helpers: `#requireRoom`/`#requireParticipant` (throw a clear `Error` on missing channel/participant so the gateway gets a deterministic failure rather than an undefined deref) and a `#getOrCreateParticipant(channelId, participantId)` for the lazy create path.
**Diff shape:**
- Add: the full `VoiceSfu` class, the inline `VoiceRoom`/`Participant` types, the `opusCodec` constant, and exported type aliases for the returned shapes (`TransportParams`, `ConsumeParams`).

## New Types / Schemas / Contracts

All in `server/src/voice/sfu.ts`. mediasoup's own types come from `mediasoup` (`import type { types } from "mediasoup"` — e.g. `types.Worker`, `types.Router`, `types.WebRtcTransport`, `types.Producer`, `types.Consumer`, `types.RtpCapabilities`, `types.RtpParameters`, `types.DtlsParameters`, `types.IceParameters`, `types.IceCandidate`).

```ts
// Direction of a participant's WebRtcTransport.
type TransportDirection = "send" | "recv";

// Returned by createTransport — the client mediasoup-client Device needs these to
// connect its transport. Relayed verbatim by the gateway (story 003).
interface TransportParams {
  id: string;
  iceParameters: types.IceParameters;
  iceCandidates: types.IceCandidate[];
  dtlsParameters: types.DtlsParameters;
}

// Returned by consume — the params the client uses to build its receiving consumer.
interface ConsumeParams {
  id: string;            // consumer id
  producerId: string;    // the remote producer being consumed
  kind: types.MediaKind; // always "audio" in M4
  rtpParameters: types.RtpParameters;
}

// Internal (not exported): per-participant SFU state.
interface Participant {
  id: string;            // gateway-chosen participant id (per socket; story 003)
  channelId: number;
  sendTransport: types.WebRtcTransport | null;
  recvTransport: types.WebRtcTransport | null;
  producer: types.Producer | null;
  consumers: Map<string, types.Consumer>; // keyed by remote producer id
}

// Internal (not exported): one voice channel's room.
interface VoiceRoom {
  channelId: number;
  participants: Map<string, Participant>;
}

// The single negotiated codec (SPEC §11).
const opusCodec: types.RtpCodecCapability = {
  kind: "audio",
  mimeType: "audio/opus",
  clockRate: 48000,
  channels: 2,
};

class VoiceSfu {
  constructor(config: Config);
  init(): Promise<void>;
  getRtpCapabilities(): types.RtpCapabilities;
  createTransport(channelId: number, participantId: string, direction: TransportDirection): Promise<TransportParams>;
  connectTransport(channelId: number, participantId: string, direction: TransportDirection, dtlsParameters: types.DtlsParameters): Promise<void>;
  produce(channelId: number, participantId: string, rtpParameters: types.RtpParameters): Promise<{ producerId: string }>;
  consume(channelId: number, participantId: string, producerId: string, rtpCapabilities: types.RtpCapabilities): Promise<ConsumeParams | null>;
  resumeConsumer(channelId: number, participantId: string, producerId: string): Promise<void>;
  pauseProducer(channelId: number, participantId: string): void;
  resumeProducer(channelId: number, participantId: string): void;
  listProducers(channelId: number, exceptParticipantId?: string): { participantId: string; producerId: string }[];
  closeParticipant(channelId: number, participantId: string): { roomEmpty: boolean };
  close(): Promise<void>;
}
```

`participantId` is a `string` chosen by the gateway in story 003 (per-socket, so a multi-socket user is multiple participants — see feature edge cases). This story does not assign ids; it only keys on them.

## API / Interface Changes

No HTTP/WS/CLI surface changes in this story (the gateway consumes the service in story 003). The new surface is the in-process `VoiceSfu` class API and the `buildApp` signature change:

| Surface | Identifier | Request / Input | Response / Output | Notes |
| ------- | ---------- | --------------- | ----------------- | ----- |
| Function | `buildApp(config)` | `Config` | `Promise<FastifyInstance>` | Now **async** (was sync). All callers must `await`. |
| Class method | `VoiceSfu.getRtpCapabilities()` | — | `types.RtpCapabilities` | Throws if not initialized. |
| Class method | `VoiceSfu.createTransport(channelId, participantId, direction)` | ids + `"send"\|"recv"` | `Promise<TransportParams>` | Lazily creates room+participant. |
| Class method | `VoiceSfu.connectTransport(...)` | + `dtlsParameters` | `Promise<void>` | Completes DTLS. |
| Class method | `VoiceSfu.produce(channelId, participantId, rtpParameters)` | client `rtpParameters` | `Promise<{ producerId }>` | Replaces any existing producer (idempotent). |
| Class method | `VoiceSfu.consume(channelId, participantId, producerId, rtpCapabilities)` | client caps | `Promise<ConsumeParams \| null>` | `null` if `!canConsume`; consumer created `paused`. |
| Class method | `VoiceSfu.resumeConsumer(...)` | ids | `Promise<void>` | Post-handshake resume. |
| Class method | `VoiceSfu.pauseProducer / resumeProducer(channelId, participantId)` | ids | `void` | Backs mute. |
| Class method | `VoiceSfu.listProducers(channelId, exceptParticipantId?)` | channel + optional exclude | `{ participantId, producerId }[]` | For fan-out wiring. |
| Class method | `VoiceSfu.closeParticipant(channelId, participantId)` | ids | `{ roomEmpty: boolean }` | Closes transports/producer/consumers; releases empty room. |
| Class method | `VoiceSfu.close()` | — | `Promise<void>` | App `onClose` teardown; idempotent. |

The authoritative version of this table lives in `contracts/sfu-core.md` (Step 7).

## Edge Cases & Gotchas

- **`buildApp` async ripple** — all call sites must await; covered in Steps 3–5 (grep audit in Step 5).
- **`init()` called twice / `close()` before `init()`** — both idempotent: `init` returns early if worker exists; `close` no-ops missing worker/router/rooms. Step 2.
- **Re-join / double join (no two mic tracks)** — `createTransport` replaces an existing transport for the same direction; `produce` closes any prior producer before creating a new one. Step 2.
- **Last participant leaves → no leaked transports** — `closeParticipant` closes both transports (cascading to producer/consumers) and deletes the room when empty, returning `roomEmpty: true` (the transition signal mirroring `PresenceRegistry.lastOffline`). Step 2.
- **Newcomer + existing call, two-way audio** — `listProducers(channelId, exceptSelf)` lets the gateway have the newcomer consume all existing producers; the producer-id-keyed consumer map lets others consume the newcomer. Step 2.
- **`canConsume` false (incompatible caps)** — `consume` returns `null` rather than throwing, so the gateway can skip that pair cleanly. Step 2.
- **Paused-consumer handshake** — consumers are created `paused: true`; `resumeConsumer` is the explicit resume the gateway/client trigger after the client side is ready. Step 2.
- **Unknown channel/participant** — `connectTransport`/`produce`/`consume` throw a clear error via `#requireParticipant`; `closeParticipant`/`pause`/`resume`/`listProducers` are tolerant no-ops/empties so teardown is always safe. Step 2.
- **Worker bind failure (RTC range unavailable)** — `init()` rejects loudly; `buildApp`'s `await sfu.init()` propagates so the server fails fast at boot rather than half-up. Steps 2–3.
- **Worker death mid-run** — `worker.on("died")` logs; single-worker design relies on process restart (no auto-respawn — out of scope for ≤10 clients). Step 2.
- **Docker prebuilt worker (glibc)** — must stay on `node:24-bookworm-slim`, not alpine; verified + commented in Step 6.
- **mediasoup type imports under NodeNext ESM** — relative imports need `.js`; mediasoup is a package import (no extension); use `import type { types } from "mediasoup"`. Resolve in Step 8.
